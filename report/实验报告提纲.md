# 《操作系统》课程设计实验报告提纲

## 一、课程设计目的

本课程设计围绕操作系统核心原理，使用 C++ 实现处理机调度、内存管理、进程同步与文件系统模拟，加深对资源管理、并发控制和系统级程序设计方法的理解。

## 二、开发环境

- 操作系统：Linux 或 Windows MinGW
- 编程语言：C++17
- 编译工具：g++ / Makefile
- 线程库：C++ 标准线程库

## 三、系统总体设计

系统采用模块化设计，主程序提供统一菜单入口，各模块分别完成独立实验：

- `scheduler.cpp`：处理机调度算法
- `memory.cpp`：动态分区管理与页面置换
- `sync_demo.cpp`：经典同步问题
- `vfs.cpp`：简易文件系统

## 四、处理机调度模块

### 设计内容

实现 FCFS、SJF、RR、Priority 四种算法。每个进程包含进程号、到达时间、服务时间和优先级。

### 输出指标

- 运行顺序/Gantt 图
- 完成时间
- 周转时间
- 等待时间
- 平均周转时间
- 平均等待时间

### 分析要点

FCFS 简单但可能导致短作业等待；SJF 可降低平均等待时间但可能造成长作业饥饿；RR 公平性较好，适合分时系统；优先级调度能体现紧急程度但需考虑低优先级饥饿。

## 五、内存管理模块

### 动态分区

实现首次适应、最佳适应和最坏适应算法。分配时按算法选择空闲分区，空间大于申请值时进行切分；回收时与相邻空闲分区合并。测试数据应构造多个可满足申请的空闲分区，以体现 FF、BF、WF 的最终分区差异。

### 页面置换

实现 FIFO 和 LRU。每次页面访问后输出内存块状态，并统计缺页次数和缺页率。

### 分析要点

FF 查找速度较快，但可能在低地址产生碎片；BF 倾向选择最小合适分区，容易产生小碎片；WF 优先切分最大空闲区，可能更快消耗大块连续空间；FIFO 实现简单但不考虑访问局部性；LRU 更符合程序局部性原理。

## 六、进程同步与并发控制模块

### 生产者-消费者

使用计数信号量控制空槽和产品数量，使用互斥锁保护缓冲区。

### 读者-写者

允许多个读者并发读取，写者写入时独占资源，并设置等待写者优先以减少写者饥饿。

### 哲学家进餐

使用服务员信号量限制最多四名哲学家同时竞争叉子，并使用 `std::lock` 同时申请左右叉，避免死锁。

## 七、文件系统模块

### 设计内容

模拟固定大小磁盘块，使用位图管理空闲块，使用目录项记录文件名、文件大小和占用块号。

### 支持操作

- 创建文件
- 写文件
- 读文件
- 删除文件
- 显示目录
- 显示空闲空间位图

- `memory_page_replacement.svg`：FIFO/LRU 缺页次数与缺页率
- `sync_timeline.svg`：同步事件时间线

增强后的基础实验还可补充以下内容：

- 处理机调度增加 SRTF 和抢占式优先级调度，对比 6 种算法的平均周转、平均等待、平均响应、平均带权周转。

- 文件系统实验可使用 `filesystem_allocation_methods.svg` 和 `filesystem_bitmap.svg` 展示分配方式与空闲块位图变化。

## 九、总结

本项目完成了基础必做部分要求。通过模拟实现可以看到，操作系统的核心任务是对处理机、内存、文件与并发资源进行合理管理；不同算法在公平性、效率、实现复杂度和资源利用率之间存在取舍。
